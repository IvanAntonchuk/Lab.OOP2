# Глосарій проєкту "Колекціонер посилань" (Link Manager)

---

* **Посилання (Link / LinkData)** — базова сутність системи, що представляє збережений веб-ресурс. Містить саму URL-адресу, автоматично або вручну заданий заголовок (title), опис, а також може належати до певної папки та мати асоційовані теги.

* **Папка (Folder)** — сутність для ієрархічного групування посилань. Кожне посилання може належати лише до однієї папки. Використовується для базової організації збережених ресурсів.

* **Тег (Tag)** — ключове слово або мітка для додаткової класифікації. Одне посилання може мати багато тегів, що дозволяє гнучкіше організовувати ресурси порівняно з папками.

* **Менеджер посилань (Link Manager)** — центральний компонент бізнес-логіки програми, який відповідає за управління колекцією: зберігання в пам'яті, додавання, видалення, редагування посилань, а також управління доступними папками та тегами.

* **Серіалізатор (Link Serializer)** — компонент, що відповідає за постійне зберігання даних. Забезпечує конвертацію об'єктів посилань у текстовий формат (JSON) для запису у файл при закритті програми та зворотне перетворення (десеріалізацію) при запуску.

* **Мережеві утиліти (Web Utils)** — допоміжний модуль, який виконує HTTP-запити до вказаної URL-адреси (парсинг веб-сторінок) для автоматичного вилучення метаданих сайту, таких як заголовок (Title) та опис.

* **QR-генератор** — модуль, що перетворює URL-адресу обраного посилання у графічний QR-код для зручного сканування камерою мобільного пристрою.

* **Фільтрація (Filtering)** — процес відображення в інтерфейсі лише тих посилань, які відповідають заданим користувачем критеріям (пошук за текстовим запитом, вибір конкретної папки або перетину тегів).

* **Експорт / Імпорт** — функціонал збереження всієї бази посилань (або її частини) у зовнішній файл та завантаження колекції з файлу для резервного копіювання чи перенесення даних на інший пристрій.

* **Графічний інтерфейс користувача (GUI / Main Window)** — візуальна оболонка застосунку, яка взаємодіє з користувачем через діалогові вікна (додавання посилання, управління папками, налаштування фільтрів тощо) та передає команди до бізнес-логіки.
